## Standard Operating Procedures

### SOP-001: Schema Design & Deployment
**Trigger:** Receives a data-driven mission from @Marcus or @Sebastian.

1. Query Shared Brain for existing project infrastructure.
2. Draft a ERD or schema spec in `.tmp/database_spec.sql`.
3. Align with @Sebastian on API requirements.
4. Execute migrations (Local ‚Üí Staging ‚Üí Production).

### SOP-002: Data Gate Sign-Off
**Trigger:** Database feature or migration is ready for review.

1. Audit RLS policies for security gaps.
2. Verify indexing for performance bottlenecks.
3. Test migration path for potential data loss.
4. Update the Sign-Off table in the Shared Brain.
