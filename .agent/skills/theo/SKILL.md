## Standard Operating Procedures

### SOP-001: Deadline Registration
**Trigger:** When a new mission or project is initialized.
1. Create a `deadlines.json` entry in `.agent/memory/`.
2. Identify and link dependencies.
3. Set alert thresholds (T-14d, T-7d, T-24h).
4. Post registry summary to chatroom.

### SOP-002: Sprint Review Prep
**Trigger:** Every Thursday at 17:00 UTC.
1. Scan `task-history.json` for the current week.
2. Compile "Velocity Report" (Completed vs Planned).
3. Identify "Drift Items" (tasks exceeding 2x original estimate).
4. Generate Mission Briefing template for @Marcus.

## Feedback Loop

### Before Every Task
```
1. Query Shared Brain: What's the current project state?
2. Check recent work: Any related tasks completed recently?
3. Review dependencies: Who else is working on adjacent items?
```

### After Every Task
```
1. Record outcome: Document what was delivered
2. Document friction: Note any blockers or inefficiencies
3. Capture learning: What would I do differently?
4. Propagate: Share findings with relevant agents
5. Update status: Mark quality gates as passed/failed
```

### Learning Capture Template
```
TASK: [Description]
OUTCOME: [Success/Partial/Failed]
ROOT CAUSE: [If issue encountered]
FIX APPLIED: [What was done]
PROPAGATE TO: [Relevant agents]
```
