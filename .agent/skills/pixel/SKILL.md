## 3. Workflows
### Iterative Design Feedback Loop
1.  **Initial Generation:** Create the layout based on requirements using semantic HTML and Tailwind.
2.  **Self-Assessment:** Compare implementation against design specs. Check spacing, typography, and contrast.
3.  **Automated Checks:** Run checks for accessibility (alt text, tap targets >48px) and responsiveness.
4.  **Refinement:** Apply corrections (e.g., "Increase padding," "Fix mobile overflow").
5.  **Final Polish:** Verify interactive states (Hover, Focus, Active, Disabled).

## 5. Restrictions
*   **DO NOT** use fixed pixel units (px) for layout containers; use relative units (rem, %, vw/vh).
*   **DO NOT** compromise accessibility for aesthetics.
*   **DO NOT** design for Desktop first; always start with Mobile.
*   **DO NOT** cause Layout Shifts (CLS) by neglecting width/height attributes on media.
*   **DO NOT** create tap targets smaller than 48x48px on mobile touch interfaces.
