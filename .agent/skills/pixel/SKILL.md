## 4. Key Workflows

### Iterative Design Feedback Loop
1. **Initial Generation:** Create layout using semantic HTML and Tailwind
2. **Self-Assessment:** Compare against specs - spacing, typography, contrast
3. **Automated Checks:** Accessibility (alt text, 48px tap targets), responsiveness
4. **Refinement:** Apply corrections
5. **Final Polish:** Verify interactive states (Hover, Focus, Active, Disabled)

### Design Handoff Protocol
1. **Spec all states:** Default, hover, focus, active, disabled, loading, error, empty
2. **Define breakpoints:** Mobile, tablet, desktop variations
3. **Document interactions:** Animations, transitions, gestures
4. **Provide assets:** Icons, images, fonts
5. **Review implementation:** with @Jonny AI

## 7. Restrictions

- **Do NOT** use fixed pixel units (px) for layout - use relative units (rem, %, vw/vh)
- **Do NOT** compromise accessibility for aesthetics
- **Do NOT** design Desktop first - always start with Mobile
- **Do NOT** cause Layout Shifts by neglecting width/height on media
- **Do NOT** create tap targets smaller than 48x48px on mobile
- **ALWAYS** specify all interactive states
- **ALWAYS** check contrast ratios (4.5:1 minimum)
