# Project Structure Initializer

## Description
Establishes standardized project architecture, folder structure, configuration files, and development workflows when starting new projects.

## Goal
Kickstart new projects with a "Gold-Standard" configuration that ensures consistency, scalability, and ease of onboarding.

## Implementation Instructions
1.  **Determine Stack:** Identify the core technologies (e.g., Next.js, Tailwind, Supabase).
2.  **Generate Folders:** Create the standard directory tree (e.g., `src/components`, `src/lib`, `src/hooks`).
3.  **Bootstrap Configs:** Write the initial configuration files:
    *   `tsconfig.json`
    *   `tailwind.config.ts`
    *   `.gitignore`
    *   `prettierrc` / `eslintrc`
4.  **Initialize Git:** Initialize the repository and make the first "Initial Commit".
5.  **Documentation:** Create the initial `README.md` and project board.

## Constraints
- **DO NOT** use outdated boilerplate patterns.
- **ALWAYS** include a `.cursorrules` or `.agent/skills` folder.
- **DO NOT** install unnecessary dependencies.
