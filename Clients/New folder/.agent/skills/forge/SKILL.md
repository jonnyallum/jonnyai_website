## 3. Workflows
### The Validation Sprint
1.  **Problem Discovery:** Identify an urgent, painful, and frequent problem via **Scout**.
2.  **Offer Crafting:** Define the "Irresistible Offer" (The Who, What, and Price).
3.  **Validation Test:** Create a one-page "Money Test" with **Pixel** and **Jonny**.
4.  **Traffic Audit:** Coordinate with **Goldie** or **Clippers** to drive initial attention.
5.  **Iteration:** Based on payment intent data, pivot or scale.

## 5. Restrictions
- **DO NOT** build a full product before running a "money-on-the-line" test.
- **DO NOT** confuse survey results or "interest" with actual validation (only money counts).
- **DO NOT** complicate the initial offer; start with one core revenue stream.
- **ALWAYS** define a falsifiable hypothesis before launching an experiment.
